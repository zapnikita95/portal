# Roadmap Портала

Краткий план развития. Детальные задачи можно вести в отдельном репозитории `portal-roadmap` или в GitHub Issues основного репо.

## Сделано (частично или полностью)

- Общий буфер (текст, файлы, картинки), несколько IP, хоткеи Win/Mac.
- Пароль сети (shared secret) в первом JSON запроса.
- Пользовательский файл для виджета (GIF/картинка), режимы auto / animated / static.
- **Имена пиров** на каждом ПК отдельно (`IP Имя` / `IP, Имя` в списке).
- **Импульс виджета** при приёме файла (показ ~3 с, если виджет был скрыт).
- **Android:** [`portal-android/`](portal-android/) — Share Sheet (`intent_filters.xml` + `android_share.py`), выбор пира, `finish()`, Docker/CI для APK; опционально **Quick Settings tile** — ещё нет.

## В работе / дальше

1. **Android:** Quick Settings tile; при желании — release-подпись и иконка в сторе.
2. **Сборки:** PyInstaller (`.exe`, `.app`) и Nuitka — скрипты и CI; подпись кода (Win/Mac) отдельно.
3. **Обновления:** проверка релизов GitHub, диалог «есть новая версия».
4. **Большие файлы:** прогресс-бар, возобновление передачи, поднимаемый лимит.
5. **Усиление auth:** challenge-response или HMAC поверх shared secret (по необходимости).
6. **Онбординг:** мастер первого запуска (пароль, IP, права Mac).
