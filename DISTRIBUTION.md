# Как отдавать Portal пользователям (Mac / Windows)

Цель: человек **скачал → запустил**, без Python и терминала.

## Уже есть: GitHub Releases

После тега `v*` (например `v0.4.0`) workflow **Portal Desktop Build** кладёт в релиз:

| Файл | Что внутри |
|------|------------|
| **Portal-macOS.zip** | `Portal.app` — распаковать, перетащить **Portal.app** в «Программы», запускать как обычную программу. |
| **Portal-Windows.zip** | папка **Portal** с `Portal.exe` — распаковать куда удобно, запускать **Portal.exe** (лучше не разбрасывать файлы по папкам). |

Ссылка для пользователей: **Releases** репозитория на GitHub → последний тег **Portal Desktop v…** → скачать ZIP под свою ОС.

### Mac: «не открывается из интернета»

Пользователь один раз в терминале:

```bash
xattr -dr com.apple.quarantine /Applications/Portal.app
```

(или путь к `.app`, если не в Программах.) Подробнее: [BUILD_DESKTOP.md](BUILD_DESKTOP.md).

### Windows

При первом запуске SmartScreen может спросить подтверждение — «Подробнее» → «Выполнить в любом случае», если доверяете источнику.

## Если нужен именно «установщик»

Сейчас формат **ZIP = достаточно для “как приложение”** (на Mac это нормальная схема для не‑App Store софта).

Дальше — опционально, больше работы:

### macOS

- **DMG** (окно «перетащи в Программы»): утилиты вроде `create-dmg`, или вручную в Дисковой утилите. Можно положить в релиз рядом с ZIP.
- **Подпись и нотаризация** — нужны **Apple Developer Program** ($/год), иначе на чужих Mac будут те же предупреждения безопасности.

### Windows

- **Inno Setup** / **NSIS** / **WiX**: собираете **один .exe установщик**, который копирует папку `Portal` в `Program Files` и создаёт ярлык.
- Локально: после `pyinstaller` взять содержимое `dist/Portal` как источник для скрипта установщика.

Имеет смысл добавлять установщик в CI только если готовы поддерживать скрипты сборки; для небольшого проекта часто хватает **одного ZIP + короткой инструкции** в описании релиза.

## Кратко для README / лендинга

1. Открыть вкладку **Releases** своего репозитория на GitHub (обычно `https://github.com/<владелец>/portal/releases`).  
2. Скачать **Portal-macOS.zip** или **Portal-Windows.zip**.  
3. Mac: распаковать → перенести `Portal.app` в «Программы».  
4. Windows: распаковать → запустить `Portal.exe` из папки `Portal`.
