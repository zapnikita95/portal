# Portal на Mac: хоткеи и виджет

На **Windows** глобальные сочетания обычно заводятся с первого раза. На **macOS** система отдельно спрашивает разрешения у **конкретного приложения**.

## Что нужно для Cmd+Ctrl+P / C / V из любого приложения

1. **Системные настройки → Конфиденциальность и безопасность → Мониторинг ввода**  
   Включи **Portal** (если запускаешь `Portal.app`).  
   Если запускаешь из **Терминала** или **IDE** — добавь туда **Python** или **Terminal** / **Cursor**.

2. Там же при необходимости: **Универсальный доступ** — тот же список (Portal или Python).

3. После изменения списка **полностью закрой Portal** (Cmd+Q) и открой снова — иначе macOS может не подцепить права.

4. В журнале может быть **`CGEventTap не создан`** — это **не приговор**: следующим шагом helper поднимает **NSEvent**. Если видишь **`✅ Глобальные хоткеи: NSEvent`** — глобальные сочетания **снаружи** окна Portal уже работают (Cmd+Ctrl+P/C/V).

5. **После каждой пересборки `Portal.app`** macOS иногда ведёт себя так, будто права «забылись»: открой **Мониторинг ввода** → **выключи** переключатель у Portal → **включи снова** → Cmd+Q Portal → запусти новый `.app`. Это не «кэш PyInstaller», а привязка TCC к бинарнику.

6. **Дополнительно ничего ставить не нужно** — только системные разрешения. Helper шлёт `t`/`c`/`v` в родительский процесс через **pipe** + опрос/fileevent.

7. Сборка **`.app`**: после правок кода нужна **новая сборка PyInstaller** — внутри лежит старый код, пока не пересоберёшь.

## Снаружи окна vs внутри окна Portal

- **Другое приложение в фокусе** — клавиши ловит **helper** (CGEventTap или NSEvent) → pipe → Portal.  
- **Фокус внутри Portal** — **Tk bind_all** (и русские з/с/м из [коммита с раскладкой](https://github.com/zapnikita95/portal/commit/7dacbe6f70979c4fa8e6733a413120c14c61fd25)).  
  Опционально: `PORTAL_MAC_NSLOCAL_MONITOR=1` — экспериментальный NSEvent local (на Python **3.13** давал **краш** при хоткее в связке с Tk — по умолчанию **выкл**).

## Краш при нажатии хоткея (Python 3.13)

Если в отчёте `PyEval_RestoreThread` / `PythonCmd` — обновись до версии, где `portal_thread_log` **не** делает `after(0)` из главного потока, и **не** включай `PORTAL_MAC_NSLOCAL_MONITOR` без необходимости.

## Если глобально не ловит, но в окне Portal работает

Глобальная часть — helper; без прав к его бинарнику остаётся только режим «внутри окна».

## Автопроверка без круга «попробуй ещё раз»

Из корня репозитория:

```bash
python3 scripts/verify_mac_hotkey_helper.py
```

То же для **собранного** приложения (те же права, что в жизни):

```bash
python3 scripts/verify_mac_hotkey_helper.py --bundle dist/Portal.app
```

Скрипт сам проверяет, поднялся ли **CGEventTap** или **NSEvent global monitor**. Нажать Cmd+Ctrl+P за тебя он не может — если ИТОГ `OK`, остаётся **один** ручной шаг: переключиться в другое приложение и нажать сочетание, в журнале Portal должна появиться строка про глобальный хоткей.

## Переменные окружения (редко нужно)

- `PORTAL_MAC_HOTKEY_LEGACY=1` — старые сочетания (Cmd+Option+P и т.д.).
- `PORTAL_MAC_NO_HOTKEY_HELPER=1` — отключить глобальный helper (только хоткеи при фокусе).
