# Portal на Mac: хоткеи и виджет

На **Windows** глобальные сочетания обычно заводятся с первого раза. На **macOS** система отдельно спрашивает разрешения у **конкретного приложения**.

## Что нужно для Cmd+Ctrl+P / C / V из любого приложения

1. **Системные настройки → Конфиденциальность и безопасность → Мониторинг ввода**  
   Включи **Portal** (если запускаешь `Portal.app`).  
   Если запускаешь из **Терминала** или **IDE** — добавь туда **Python** или **Terminal** / **Cursor**.

2. Там же при необходимости: **Универсальный доступ** — тот же список (Portal или Python).

3. После изменения списка **полностью закрой Portal** (Cmd+Q) и открой снова — иначе macOS может не подцепить права.

4. В журнале может быть **`CGEventTap не создан`** — это **не приговор** и **не обвинение**: у многих в «Мониторинг ввода» всё включено, а `CGEventTapCreate` всё равно возвращает пусто (новая сборка `.app`, другой путь к `python3`, кэш TCC). Следующим шагом helper поднимает **NSEvent**. Если видишь **`✅ Глобальные хоткеи: NSEvent`** — опирайся на него; проверка — Cmd+Ctrl+P **из другого приложения** и строка **`Глобальный хоткей`** в журнале.

### Права уже ON на скрине, но в логе снова «CGEventTap не создан»

1. В **`portal_hotkey_debug.log`** (или в файле активности) найди строку **`i hotkey-helper exe …`** — это **точный путь бинарника** дочернего процесса. В «Мониторинг ввода» должна быть запись **именно для него** (не «похожий» python из другого каталога).
2. После **каждой новой сборки** `Portal.app`: открой список → **выключи** переключатель у Portal → **включи снова** (или удали строку и **[+]** после следующего запуска). Подпись приложения меняется — macOS иногда держит старую привязку.
3. Переустановка из **DMG** сама по себе не «чинит» TCC, если не сделать шаг 2. Имеет смысл ставить **свежую** сборку в **Программы** и один раз обновить переключатель.
4. **Не путать** разделы: «Универсальный доступ» и «Мониторинг ввода» — разные списки; для глобальных клавиш критичен **Мониторинг ввода** (+ иногда оба).

5. **После каждой пересборки `Portal.app`** macOS иногда ведёт себя так, будто права «забылись»: открой **Мониторинг ввода** → **выключи** переключатель у Portal → **включи снова** → Cmd+Q Portal → запусти новый `.app`. Это не «кэш PyInstaller», а привязка TCC к бинарнику.

6. **Дополнительно ничего ставить не нужно** — только системные разрешения. Helper шлёт `t`/`c`/`v` в родительский процесс через **pipe** + опрос/fileevent.

7. Сборка **`.app`**: после правок кода нужна **новая сборка PyInstaller** — внутри лежит старый код, пока не пересоберёшь.

## Снаружи окна vs внутри окна Portal

- **Другое приложение в фокусе** — клавиши ловит **helper** (CGEventTap или NSEvent) → pipe → Portal.  
- **Фокус в главном окне Portal** — **Tk bind_all** + на macOS дополнительно **физические keycode** (35/8/9 для P/C/V при Cmd+Ctrl), если русская раскладка не даёт сработать `<Command-Control-з>` и т.п.  
- **Фокус в окне «Журнал»** (многострочное поле) — у `tkinter.Text` свои привязки **раньше**, чем тег `all`, поэтому раньше `bind_all` не срабатывал; в актуальной версии те же сочетания дублируются на внутренний `Text` журнала.  
- **Русские з/с/м** — см. [коммит с раскладкой](https://github.com/zapnikita95/portal/commit/7dacbe6f70979c4fa8e6733a413120c14c61fd25).  
  **Python 3.13+:** **NSEvent local monitor по умолчанию выключен** (реентрантность Tcl↔AppKit при перехвате клавиш внутри окна давала SIGABRT). Работают **Tk bind_all** и физические keycode. Включить local monitor только при необходимости: `PORTAL_MAC_NSLOCAL_MONITOR=1`.

### Порядок helper и конфликт с другими программами (Whisper и т.д.)

- По умолчанию **сначала NSEvent global**, затем **CGEventTap** — так чаще стабильнее в Chrome при «занятом» CGEventTap другим приложением. Вернуть старый порядок: `PORTAL_MAC_HOTKEY_CG_FIRST=1`.
- Доп. попытка tap: `PORTAL_MAC_HOTKEY_TRY_HID_TAP=1` (HID session вместо только session tap).
- Два глобальных перехватчика (Portal + Whisper/pynput) могут мешать друг другу — см. `whisper/PORTAL_AND_WHISPER_MAC.md`: у Whisper задай сочетание **без ⌘** или останови клиент на время проверки Portal.

## Краш при нажатии хоткея (Python 3.13)

Если в отчёте `PyEval_RestoreThread` / `PythonCmd` — обновись до версии, где `portal_thread_log` **не** делает `after(0)` из главного потока, и **не** включай `PORTAL_MAC_NSLOCAL_MONITOR` без необходимости.

## Краш сразу после старта (Python 3.13)

Если в стеке есть **`-[NSRunLoop runUntilDate:]`** вместе с **Tk** / **`Tcl_DoOneEvent`** / **`_tkinter_tkapp_mainloop`** — это **нельзя** совмещать: отдельный поток с `NSRunLoop` внутри Tk `after()` давал **SIGABRT**. В актуальной версии глобальные хоткеи на 3.13 снова только через **hotkey-helper subprocess** + pipe (без кручения NSRunLoop из Tk).

## Если глобально не ловит, но в окне Portal работает

Глобальная часть — helper; без прав к его бинарнику остаётся только режим «внутри окна».

## Автопроверка без круга «попробуй ещё раз»

Из корня репозитория:

```bash
python3 scripts/verify_mac_hotkey_helper.py
```

То же для **собранного** приложения (те же права, что в жизни):

```bash
python3 scripts/verify_mac_hotkey_helper.py --bundle dist/Portal.app
```

Скрипт сам проверяет, поднялся ли **CGEventTap** или **NSEvent global monitor**. Нажать Cmd+Ctrl+P за тебя он не может — если ИТОГ `OK`, остаётся **один** ручной шаг: переключиться в другое приложение и нажать сочетание, в журнале Portal должна появиться строка про глобальный хоткей.

## Переменные окружения (редко нужно)

- `PORTAL_MAC_HOTKEY_LEGACY=1` — старые сочетания (Cmd+Option+P и т.д.).
- `PORTAL_MAC_NO_HOTKEY_HELPER=1` — отключить глобальный helper (только хоткеи при фокусе).
