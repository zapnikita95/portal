# 🌀 Портал

Приложение для передачи файлов и синхронизации буфера обмена по **локальной сети** и **mesh-VPN** (Tailscale, NetBird, ZeroTier, Headscale и т.п.) с красивым UI в стиле портала и виджетом на рабочем столе.

→ [Как подключаться: LAN, mesh-VPN, mDNS](docs/CONNECTING.md)

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Python 3.8+](https://img.shields.io/badge/python-3.8+-blue.svg)](https://www.python.org/downloads/)

## ✨ Возможности

- 📤 **Передача файлов** между устройствами в Tailscale сети
- 📋 **Общий буфер**: текст, скриншот/картинка и **скопированные файлы** (Finder/Проводник) — на принимающем ПК можно сразу **Ctrl+V / Cmd+V**
- 🎨 **Современный UI** в темной теме
- 🌀 **Виджет-портал на рабочем столе** с анимацией как в игре Portal; в настройках можно указать **свой GIF/PNG/JPEG** и режим (авто / анимация / статика с масштабом при открытии)
- 🖱️ **Drag & Drop** - просто перетащите файл в портал
- ⌨️ **Горячие клавиши**:
  - Показать/скрыть виджет: `Ctrl+Alt+P` (Win) / по умолчанию **`Cmd+Ctrl+P`** (Mac); старые: `PORTAL_MAC_HOTKEY_LEGACY=1` → `Cmd+Option+P`. На русской раскладке Mac те же физические клавиши: **`Cmd+Ctrl+з / с / м`**
  - **Общий буфер**: отправить — `Ctrl+Alt+C` (Win) / **`Cmd+Ctrl+C`** (Mac; legacy: `Cmd+Shift+C`)
  - **Общий буфер**: забрать с первого отмеченного IP — `Ctrl+Alt+V` (Win) / **`Cmd+Ctrl+V`** (Mac; legacy: `Cmd+Shift+V`). Один скопированный файл на удалённом ПК передаётся по сети (до ~100 МБ).  
    *(на принимающих ПК должен быть запущен «Запустить портал»; в главном окне — список IP и галочки получателей)*
- 🖱️ **Виджет**: перетаскивание окна — **Alt + левую кнопку**; выбор файла — **Ctrl + клик**; сменить IP — **двойной клик**
- 💾 **Несколько IP**: список в главном окне → «Сохранить список IP», затем **галочки — кому слать сразу** → «Сохранить выбор». Файлы и буфер уходят на **все отмеченные** машины. Строка пира: **`100.x.x.x`** или **`100.x.x.x Имя`** / **`100.x.x.x, Имя`** — имя только на этом компьютере (в конфиге `peer_aliases`). Конфиг: `%APPDATA%\Portal\config.json` (Win) или `~/Library/Application Support/Portal/config.json` (Mac), поля `peer_ips`, `peer_send_targets`, `peer_aliases`
- ✨ **Приём файла:** если виджет был скрыт, он **кратко показывается (~3 с)** и снова скрывается (отключить: `PORTAL_WIDGET_PULSE_ON_RECEIVE=0`, длительность: `PORTAL_WIDGET_PULSE_SECONDS=3`)
- 🔒 **Пароль сети (shared secret)** в настройках — одинаковый на всех своих ПК; шифрование трафика даёт VPN (Tailscale и т.д.), пароль режет доступ к порту **12345** в LAN
- 🚀 **Работа без браузера** - нативное приложение
- 💿 **Сборка «как обычная программа»** (без `.command` / `.bat`): **[BUILD_DESKTOP.md](BUILD_DESKTOP.md)** — `pyinstaller -y pyinstaller_portal.spec` → **Windows:** `dist/Portal/Portal.exe`, **macOS:** `dist/Portal.app` (иконка из `portal_main.gif` (скрипт `generate_branding_icons.py`)). **В облаке:** GitHub Actions **Portal Desktop Build** (артефакты или релиз по тегу `v*`, см. BUILD_DESKTOP.md). **Как отдавать пользователям (ZIP / установщики):** [DISTRIBUTION.md](DISTRIBUTION.md).
- 📜 **История передач** (SQLite `portal_history.sqlite3` рядом с `config.json`): кнопка «История» в шапке — фильтр, **переотправить** файл на те же IP, **копировать** путь или текст.
- 🔎 **Найти локально** (фаза A): скан `/24` вокруг текущего IP — узлы с ответом Portal на `:12345`, добавление в список пиров.
- 📱 **Мобильный клиент (Flutter)** — **основной** путь: [portal_flutter/README.md](portal_flutter/README.md). Сборки: **Portal Flutter Build** → релиз `portal-flutter-latest` (`Portal-Flutter.apk`). Kivy **Portal-Android.apk** (`portal-android-latest`) оставлен **legacy** (если нужна старая сборка). **iOS:** [portal_flutter/IOS_INSTALL.md](portal_flutter/IOS_INSTALL.md).
- 🤖 **Android foreground service** (приём в фоне): [docs/ANDROID_FGS.md](docs/ANDROID_FGS.md).

## 📋 Требования

- Python 3.8+
- Windows 10/11, macOS, или Linux
- Tailscale (опционально, для работы через VPN) или локальная сеть

### 🌐 Язык интерфейса (RU / EN)

В **⚙ Настройки → Общее** выбери **Русский** или **English**, нажми «Сохранить язык» и **перезапусти** Портал. В `config.json`: `"ui_language": "ru"` или `"en"`. Тексты собраны в **`portal_i18n.py`** (два словаря в одном файле — без внешних JSON, чтобы PyInstaller не терял ресурсы). Виджет и macOS (заголовок окна, поиск NSWindow) учитывают оба языка.

## Зачем Portal и чем удобнее «облака» и мессенджеров

- **Прямая связь устройство ↔ устройство** по TCP — без загрузки в чужой сервер и без лимитов «облака».
- **Общий пароль сети (shared secret)** — лишний без пароля к твоему Порталу не подключится (первый JSON каждого запроса).
- **Tailscale / VPN** — шифрование на пути; в связке с паролем это нормальная модель для своих машин.
- **Буфер обмена + файлы + история** в одном месте, горячие клавиши, виджет на столе.

## Безопасность (кратко)

На **LAN без VPN** теоретически виден твой IP: кто-то из той же сети может **попытаться** достучаться до порта **12345**. **Пароль сети** в Portal закрывает это на уровне приложения: без совпадения `secret` запрос отклоняется. Для публичных Wi‑Fi и чужих сетей разумно **Tailscale** (или не слушать Portal вне доверенной сети / файрвол).

### Внешний вид виджета

В главном окне: блок **«Внешний вид портала»** — путь к файлу (`Обзор…`), режим отображения, плюс **размер (квадрат, px), угол экрана и отступы** (кнопка «Сохранить размер и угол»; в `config.json`: `widget_size`, `widget_corner` = `br`/`bl`/`tr`/`tl`, `widget_margin_x`, `widget_margin_y`).

- **Авто** — GIF крутится как анимация; одиночный PNG/JPEG/WebP — **статика** (картинка разворачивается при показе виджета и сворачивается при скрытии).
- **Всегда анимация** — покадрово, даже для многостраничных форматов.
- **Всегда статика** — один кадр и масштаб.

Пустой путь = как раньше, файлы из папки **`assets/`** в проекте (`portal_animated.gif` и т.д.). **Видео (MP4/WebM/MOV):** в настройках кнопки **«Обзор…»** (фильтр «Видео») или **«Видео → GIF»** — конвертация в `assets/portal_animated.gif` через `import_portal_from_mp4.py`. Прозрачный фон: см. ниже (хромакей Windows, переменные macOS).

### Пароль сети (shared secret) — быстрый слой защиты

Задай **одинаковый пароль на всех своих ПК** (⚙ Настройки → вкладка «Пароль»: «Подставить», «Сохранить», **«Сгенерировать и разослать по сети»** или **«Разослать пароль из поля»**). Рассылка идёт по TCP на все IP из вкладки «Пиры» (текущий пароль в запросе должен совпадать — как при обычной отправке файлов). На других машинах поле в настройках обновится само; в журнале будет строка о приёме.

В первом JSON каждого запроса уходит поле `secret`; приёмник сравнивает его с сохранённым значением (**без новых зависимостей**, `hmac.compare_digest`).

- **Пустой пароль** — поведение как в старых версиях (без проверки), удобно для обновления, но **не рекомендуется в публичной Wi‑Fi**.
- **`PORTAL_NO_REMOTE_SECRET_SYNC=1`** — не слать пароль по сети (только локально).
- **`PORTAL_ALLOW_LEGACY_NO_AUTH=1`** на **принимающем** ПК — временно принимать старые клиенты **без** поля `secret` (для переходного периода; в LAN это ослабляет защиту).

Позже при необходимости можно усилить схему (challenge-response, подпись сообщений, Fernet/HMAC) поверх того же пароля.

## Обновление установленного приложения

Поставь новую версию из **Releases** (поверх или заменой папки/`.app`), либо встроенный механизм обновления, если он есть в твоей сборке. Обновление через `git pull` нужно только если ты ведёшь проект как репозиторий с Python.

### macOS после скачивания из интернета

Если система ругается на приложение: `xattr -dr com.apple.quarantine /Applications/Portal.app` (или путь к твоему `.app`). См. [DISTRIBUTION.md](DISTRIBUTION.md).

**Прозрачный фон вокруг портала (macOS):** по умолчанию хромакей **магента `#FF00FF`** (не съедает тёмные края огня). Свой цвет: `export PORTAL_WIDGET_CHROMA=#010101`. Полностью отключить «дырявое» окно: `PORTAL_WIDGET_NO_MAC_TRANSPARENT=1`.

**Виджет «окошком» с рамкой (macOS, по умолчанию):** фон подложки — в **⚙ Настройки → Виджет → «Применить фон»** (или `widget_mac_panel_bg` в `config.json`). Старое поведение «хромакей на весь стол»: `PORTAL_WIDGET_FRAMED=0`.

**Приём файлов:** в настройках список **режима** — на диск и в буфер / только папка / только буфер (+ файл в папке). Отправка «как из буфера» (`portal_clipboard`) по-прежнему всегда кладёт в буфер на принимающей стороне. **Отдельная папка по IP:** вкладка «Папка и приём» — текстовое поле «IP → путь» (совпадение **строго** с IP в логе, например `100.x.x.x`); если строки для IP нет — используется общая папка. В режиме **«только в буфер»** файлы пишутся во временную папку, маппинг по IP не применяется.

**Drag & Drop файла на виджет (macOS):** включён через `tkinterdnd2`. Если на Python 3.13+ падает при старте — запуск с `PORTAL_NO_MAC_DND=1` (DnD только из главного окна).

### Запуск из клонированного репозитория (разработка)

Нужны Python 3.8+ и `pip install -r requirements.txt`. На macOS при ошибке `_tkinter` — `./fix.sh`. Запуск: `python3 portal.py` или `Portal.command` (виджет по умолчанию; `--no-widget` — без виджета).

## 📖 Использование

### Запуск портала

1. Запустите приложение на обоих устройствах
2. На устройстве-получателе нажмите "🚀 Запустить портал"
3. Приложение покажет ваш IP адрес (Tailscale IP, если доступен, или локальный IP)

### Использование виджета-портала

1. Запустите приложение (виджет **включён по умолчанию**):
   ```bash
   python portal.py
   ```
   Чтобы без виджета: `python portal.py --no-widget`

2. На рабочем столе появится анимированный портал (правый нижний угол)

3. **Перетащите файл в портал** или кликните на него для выбора файла

4. При первом использовании введите IP адрес получателя

5. Используйте горячие клавиши для показа/скрытия виджета:
   - Windows/Linux: `Ctrl+Alt+P`
   - Mac: по умолчанию `Cmd+Ctrl+P` (русская раскладка: `Cmd+Ctrl+з`); legacy: `PORTAL_MAC_HOTKEY_LEGACY=1` → `Cmd+Option+P`

6. **Двойной клик** на виджете открывает настройки IP
7. **Правый клик** открывает контекстное меню

### Своя анимация портала (MP4 → GIF)

Виджет берёт картинку из папки `assets/` (приоритет: `portal_animated.gif`).

```bash
pip install -r requirements.txt   # там есть imageio + imageio-ffmpeg
python import_portal_from_mp4.py "C:\путь\к\видео.mp4"
```

Скрипт создаст `assets/portal_animated.gif` и `portal_static.gif`. Перезапусти Портал — вместо «нарисованного» круга будет твой портал.  
*(Если в PATH есть `ffmpeg`, он используется первым; иначе — встроенный ffmpeg из `imageio-ffmpeg`.)*

### Пресеты импульса виджета (разные порталы по IP)

При коротком «импульсе» виджета (приём/отправка) можно подставить **другой GIF/WebP** в зависимости от **IP пира** и **типа события**.

1. Открой **⚙ Настройки → Виджет** — блок **«Импульс виджета»**.
2. Каждая строка: `IP` или `*` → `receive` | `receive_file` | `send` → `id_пресета` (таб или запятая как разделитель).
3. **Порядок:** сначала ищется правило для конкретного IP, затем для `*`. Для `receive_file`, если строки нет, используется правило `receive`.
4. Встроенные пресеты: `main` (как поле «Медиа»), `blue`, `yellow`, `classic`, `space`, `strange`, `rick`, `giphy` — файлы в `assets/presets/` рядом с приложением.

### macOS: `CERTIFICATE_VERIFY_FAILED` (APK / GitHub)

Запросы к GitHub (скачать APK, запустить сборку) идут через **urllib + certifi** из `requirements.txt`. Установи зависимости: `pip install -r requirements.txt`.  
Если ошибка остаётся, для Python с python.org запусти **`Install Certificates.command`** из папки установки Python, либо обнови системные корни (macOS).

### Отправка файла

1. На устройстве-отправителе нажмите "📤 Отправить файл"
2. Выберите файл
3. Введите IP адрес получателя:
   - Если используется Tailscale: введите Tailscale IP (например, `100.64.1.2`)
   - Если используется локальная сеть: введите локальный IP получателя (например, `192.168.1.100`)
4. Файл будет отправлен и сохранен в папку `Desktop/Portal_Received`

### Получение файла

- Файлы автоматически сохраняются в `Desktop/Portal_Received`
- В логе отображается информация о полученных файлах

### Отправка буфера обмена

1. Скопируйте текст в буфер обмена (Ctrl+C)
2. Нажмите "📋 Отправить буфер"
3. Введите Tailscale IP получателя
4. Буфер обмена будет автоматически скопирован на устройстве получателя

## 📝 Примечания

- **С Tailscale**: Убедитесь, что оба устройства находятся в одной Tailscale сети
- **Без Tailscale**: Устройства должны быть в одной локальной сети (Wi-Fi/LAN)
- Firewall может блокировать соединения - разрешите порт 12345 при необходимости
- При использовании локальной сети убедитесь, что устройства видят друг друга
- **Соединения в логе**: второй ПК каждые ~20 с шлёт **ping** (проверка «Портал онлайн») — это **не** передача файла. При реальной отправке в логе будет строка `🔗 … · file` и `📥 Прием файла`. На macOS после приёма Finder откроет папку с файлом (`open -R`). Отключить: `PORTAL_REVEAL_RECEIVED=0`.
- **Python 3.13 + Drag&Drop в главном окне**: патч в `portal_tk_compat.py` вызывается **после** `TkinterDnD._require(...)` — иначе методы ещё не на `BaseWidget`, и `drop_target_register` не появится на `Misc`/CTk.
- **macOS: Терминал всплывает при Cmd+Shift+C?** Запускай через `start_portal.command` — по умолчанию Портал уходит в **фон** (`nohup`), лог `~/Library/Logs/portal_nohup.log`. Передний план только для отладки: `./start_portal.command --foreground`.
- **macOS: виджет не виден / нужна магента-хромакей?** `PORTAL_MAC_CHROMA_ONLY=1 ./start_portal.command` (по умолчанию в скрипте **не** задаётся — пробуем нормальную прозрачность).
- **Интерфейс:** сверху кнопки **⚙ Настройки** (папка приёма, режим входящих, виджет, список пиров, пароль), **📥 APK**, **📋 Журнал**, **❓** (горячие клавиши). На главном экране — кому слать и (если пароль ещё не задан) баннер «Пароль сети».
- **Автозапуск приёма:** при старте приложения сервер поднимается сам. Отключить: `PORTAL_NO_AUTO_START=1`.
- **Журнал:** окно по кнопке «📋 Журнал»; «Копировать всё», Cmd/Ctrl+C в поле; дублирование в `portal_activity.log` рядом с `config.json` (macOS: `~/Library/Application Support/Portal/`).
- **Глобальные хоткеи на Mac**: в «Конфиденциальность» нужны **и** Универсальный доступ, **и** **Мониторинг ввода** (Input Monitoring) — это разные переключатели. После фонового запуска добавь в список тот `python3`, который реально в PATH (см. `which python3`).
