# 🚀 Быстрый старт

## Установка за 3 шага

1. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

2. **Запустите приложение:**
   ```bash
   python portal.py
   ```
   Или просто дважды кликните `run.bat`

   **Графический портал (виджет) по умолчанию скрыт** — показ: `Ctrl+Alt+P` (Win) / `Cmd+Ctrl+P` (Mac; `PORTAL_MAC_HOTKEY_LEGACY=1` → `Cmd+Option+P`). На русской раскладке Mac: `Cmd+Ctrl+з / с / м`.  
   Чтобы **сразу увидеть круглый портал** без хоткея (тест/отладка):
   ```bash
   python portal.py --show-portal
   ```
   Или: `set PORTAL_SHOW_ON_START=1` (Windows) перед запуском.

   Проверка, что окно реально появляется: `python test_graphic_portal_visible.py`

   При запуске через **`start_portal.command`** / **`start_portal.bat`** перед стартом автоматически выполняется **`python scripts/verify_portal_smoke.py`**. Если упало — приложение не поднимают; одноразовый пропуск: **`PORTAL_SKIP_SMOKE=1`** (Mac) или **`set PORTAL_SKIP_SMOKE=1`** (Win, затем bat).

   **После merge / перед релизом** (Mac и Windows — один `main`):
   ```bash
   python scripts/verify_portal_smoke.py
   ```
   Падает — не пушить в `main`, пока не починишь.

3. **Начните использовать:**
   - На устройстве-получателе: нажмите "🚀 Запустить портал"
   - На устройстве-отправителе: нажмите "📤 Отправить файл" или "📋 Отправить буфер"
   - **LAN / общая Wi‑Fi:** задай **одинаковый пароль сети** в настройках на всех своих ПК («Сгенерировать» → «Сохранить»). Подробно: [README — безопасность](README.md#безопасность-и-сеть).
   - Введите IP адрес получателя (показан в окне приложения)

## Пример использования

### Сценарий 1: Передача файла через Tailscale

1. **Устройство A (получатель):**
   - Запустите Портал
   - Нажмите "🚀 Запустить портал"
   - Запомните Tailscale IP (например, `100.64.1.2`)

2. **Устройство B (отправитель):**
   - Запустите Портал
   - Нажмите "📤 Отправить файл"
   - Выберите файл
   - Введите IP устройства A: `100.64.1.2`
   - Файл появится в `Desktop/Portal_Received` на устройстве A

### Сценарий 2: Синхронизация буфера обмена

1. Скопируйте текст на устройстве A (Ctrl+C)
2. На устройстве A нажмите "📋 Отправить буфер"
3. Введите IP устройства B
4. Текст автоматически скопируется в буфер обмена на устройстве B

## Буфер: «забрать с другого ПК», ошибки и настройки (кратко)

1. **Ошибка 10061 (Windows)** — текст «конечный компьютер отверг запрос» **не значит**, что кто-то нажал «отказать». Обычно на этом IP **ничего не слушает порт 12345**: не нажато «Запустить портал», неверный IP, VPN/файрвол. В журнале Портала теперь явно подсказка: запустить приём на удалённой машине и проверить адрес.

2. **«Забрать буфер»** раньше тянул с пира в основном **текст** (`pyperclip`). Если на Mac в буфере **файл** или **картинка**, ответ теперь такой же по формату, как при отправке: `clipboard` / `clipboard_files` / `clipboard_rich`, поток читается на стороне, кто запросил (**Ctrl+Alt+V** / по умолчанию **Cmd+Ctrl+V**; legacy: **Cmd+Shift+V** или **Cmd+Option+V**).

3. **Legacy (Mac):** **Cmd+Option+V** — дополнительно к **Cmd+Shift+V** для «забрать буфер». В части приложений **Cmd+Option+V** уже занят — тогда используй **Cmd+Shift+V**.

4. **Двойной pull** — второй запрос, пока первый идёт, получит сообщение «Уже выполняется запрос…».

5. **Режим приёма файлов из буфера** (только папка / только буфер / папка+буфер): в **главном окне** — блок «Куда сохранять…» и выпадающий список; если пользуешься **только виджетом** — **ПКМ по порталу** → «Приём файлов из буфера…». Под заголовком папки в окне — серая подсказка, к чему это относится.

**Важно:** оба ПК должны быть на **одной версии** кода (`git pull`). На машине, **с которой тянешь** буфер, в момент запроса на другой должен быть **запущен приём** («Запустить портал»).

6. **macOS: файлы в буфере и хоткеи**
   - **Вставить** файл, который Портал положил в буфер: в окне **Finder** (внутри папки, не «MacBook» в корне) — **Cmd+V**. Это обычная вставка macOS.
   - **Cmd+Ctrl+V** (legacy: **Cmd+Shift+V**) у Портала = **«забрать буфер с другого ПК»**, не вставка в Finder. Русская раскладка: **м** вместо **V**.
   - Чтобы **отправить** выделенный в Finder файл хоткеем: сначала **Cmd+C** в Finder, затем хоткей отправки буфера (например **Cmd+Shift+C** в legacy или **Cmd+Ctrl+C** по умолчанию). Раньше на Mac файлы из Finder не подхватывались — теперь читается системный pasteboard.
   - Если копируешь **иконку файла** (например `Фото.jpg` на Рабочем столе), должен уходить **файл**, а не «скрин превью»: иначе на другом ПК была бы **чужая** растровая картинка. Сначала читается `NSFilenamesPboardType`, и только потом Pillow.

## Решение проблем

**Проблема:** "Tailscale IP не найден"
- **Решение:** Приложение будет использовать локальный IP. Убедитесь, что устройства в одной сети.

**Проблема:** "Ошибка подключения"
- **Решение:** 
  - Проверьте, что портал запущен на получателе
  - Проверьте правильность IP адреса
  - Разрешите порт 12345 в Firewall

**Проблема:** Файлы не принимаются
- **Решение:** Проверьте, что папка `Desktop/Portal_Received` существует и доступна для записи
